## Troubleshooting

If `load_rotten_tomatoes()` fails with `TypeError: Pickler._batch_setitems() takes 2 positional arguments but 3 were given`, the issue is caused by the local `datasets/dill/pickle` stack in the `DLA2026-transformers` environment. The helper function now catches that specific error and loads the already downloaded Rotten Tomatoes Arrow files from the local Hugging Face cache.

After updating `src/dla_lab2/sentiment.py`, restart the notebook kernel before rerunning Notebook 01. Otherwise Jupyter may keep the old imported version of the function in memory.

## Methodological Choices

Exercise 1 uses DistilBERT as a frozen feature extractor and a Linear SVM as a stable baseline. This is useful because it gives a reproducible lower bound before training the transformer.

Exercise 2 fine-tunes `distilbert/distilbert-base-uncased` with a sequence classification head using Hugging Face `Trainer`, dynamic padding and Scikit-learn metrics.

Exercise 3.1 compares two efficient approaches:

- LoRA on DistilBERT attention projections;
- partial freezing of embeddings and early transformer layers.

Exercise 3.2 uses CLIP on ImageNet-Sketch. The chosen adaptation is CLIP-Adapter, not LoRA inside the transformer. This is still parameter-efficient because CLIP remains frozen and only a small MLP is trained on image features. This choice is easier to run on limited VRAM and directly targets the sketch domain shift.
